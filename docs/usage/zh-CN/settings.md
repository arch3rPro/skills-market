# 设置

## 作用

`设置` 用来配置环境级行为。

## 主要区域

- 中央仓库路径
- 默认同步模式
- 默认场景
- 工具启用状态和工具路径
- 自定义 Agent
- 自定义 Agent 同步模式
- 语言和主题
- 字号
- 代理
- SkillsMP API Key
- ClawHub API Key
- Git 备份远程配置
- WebDAV 云同步配置
- 应用更新检查

## 自定义 Agent

自定义 Agent 允许你定义内置工具之外的额外同步目标。

可配置内容包括：

- 显示名称
- Skills 路径
- 可选的项目工作区路径
- 独立同步模式

## 搜索和外部服务

`设置` 也负责管理外部服务配置：

- `SkillsMP API Key`：用于市场 AI 搜索
- `ClawHub API Key`：用于 ClawHub 集成
- `代理`：在需要时为 Git 和网络请求提供统一出口

### WebDAV 云同步

`WebDAV 云同步` 通过兼容 WebDAV 的存储服务上传和下载完整应用状态快照。

快照包含 SQLite 元数据导出和中央 Skills 目录归档。它适合不想维护 Git 远程仓库、但需要多设备同步应用状态的场景。

上传会覆盖远端快照。下载会在应用创建安全备份后覆盖本地应用数据，因此恢复前应先查看远端元数据。

首次使用流程：

1. 选择服务商预设。
2. 输入 WebDAV URL、用户名和应用密码。
3. 测试连接。
4. 保存配置。
5. 在源设备上传快照。
6. 在目标设备查看远端元数据后再下载。

## 最佳实践

尽早完成 `设置` 中的基础配置，尤其是中央仓库路径、启用工具、自定义 Agent 和 Git 远程。这样后续管理场景和同步 Skills 时会更稳定。
