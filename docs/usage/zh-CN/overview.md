# 使用总览

## 核心模型

Skills-Manager-Plus 的核心思路是：先维护一个中央技能库，再决定每个工作流、每个 Agent 和每个项目如何消费这套 Skills。

主要概念包括：

- `中央库`：所有导入技能的主存储位置。
- `场景`：一个具名工作流配置，拥有自己独立的启用技能集合。
- `已安装工具`：可以接收同步技能的支持工具。
- `项目工作区`：可以与中央库对比的项目本地技能区域。
- `自定义 Agent`：由用户定义路径和同步配置的额外工具目标。

## 典型工作流

1. 创建一个场景。
2. 通过 `技能商店` 导入 Skills。
3. 打开 `技能管理`，决定当前场景启用哪些 Skills。
4. 将这些 Skills 同步到已安装工具。
5. 如果仓库中存在本地 Skills，则使用 `项目工作区` 进行对比和流转。
6. 如果需要恢复点或多机同步，则使用 `Git 备份`。

## 主导航

- `数据看板`：快速状态和快捷入口。
- `技能管理`：中央库和本地扫描结果的主要管理界面。
- `技能商店`：从外部来源导入新 Skills。
- `场景`：切换不同工作流配置。
- `项目工作区`：检查和同步仓库本地 Skills。
- `设置`：配置路径、工具、同步、搜索 Key 和环境行为。
