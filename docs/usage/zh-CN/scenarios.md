# 场景

## 作用

`场景` 用来为不同工作流维护不同的启用技能集合。

## 为什么重要

如果没有场景，所有工作流都会共享同一套活跃 Skills，很快就会变得混乱。场景可以帮助你隔离不同上下文，例如：

- 个人开发
- 客户项目
- 学习实验
- 某个产品专属的 Agent 配置

## 这里能做什么

- 创建场景。
- 重命名场景。
- 调整场景顺序。
- 在侧边栏切换当前场景。
- 在共享中央库的同时，切换当前启用技能集合。

## 它和 Skills 的关系

- 中央库存放全部已导入 Skills。
- 场景决定当前哪些 Skills 被启用。
- 最终同步结果同时取决于当前场景和每个 Skill 的按 Agent 同步状态。

## 建议用法

按工作流边界来划分场景，而不是按细碎偏好来划分。少量清晰的场景比大量重叠场景更容易维护。
