# Git 备份

## 作用

`Git 备份` 用来保护中央库，为其提供版本历史、恢复点和多机同步能力。

## 备份的内容

应用备份的是当前中央仓库中的 `skills/` 目录。

默认路径为：

```text
~/.skills-manager-plus/skills/
```

## 基本流程

1. 在 `设置` 中保存远程仓库地址。
2. 打开 `技能管理`。
3. 执行首次备份初始化。
4. 之后通过 `同步到 Git` 持续执行拉取、提交和推送。
5. 通过 `版本历史` 查看和恢复快照。

## 恢复模型

恢复某个版本时，应用会创建新的恢复提交，而不是删除后续历史。这种方式更安全，也更便于审计。

## Git 备份和 WebDAV 云同步

`Git 备份` 和 `WebDAV 云同步` 解决的是不同问题。

`Git 备份` 为中央 Skills 文件提供版本历史和基于 Git 的同步。`WebDAV 云同步` 传输的是完整应用状态快照，包含数据库元数据和 Skills 文件。

从 WebDAV 恢复后，Git 备份可能会显示本地文件变化。应用不会自动提交或推送这些变化；请先检查，再执行 Git 同步。

## 重要说明

SQLite 数据库不会进入 Git 备份。应用将技能文件视为可持续恢复的数据源，元数据可以通过重新扫描重建。
