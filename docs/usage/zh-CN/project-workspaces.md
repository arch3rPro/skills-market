# 项目工作区

## 作用

`项目工作区` 用来处理已经包含本地 Skills 的仓库，或者需要项目级 Skill 协作的场景。

## 它能做什么

- 关联一个项目目录或外部 Skills 根目录。
- 检测项目中按 Agent 分布的本地 Skills 文件夹。
- 将项目本地 Skills 与中央库进行对比。
- 展示同步状态，例如已同步、仅项目存在、项目较新、中央库较新或双方分叉。
- 将项目本地 Skills 导入中央库。
- 将中央库 Skills 导出回项目。

## 工作区类型

- `项目工作区`：绑定一个仓库，并读取其本地 Agent Skills 路径。
- `关联工作区`：把外部 Skills 根目录作为独立工作区管理，但不参与全局场景同步。

## 典型用例

- 某个仓库已经存在本地 Claude Code 或 Codex Skills。
- 团队把项目专属 Skills 保存在版本库中。
- 你需要在导入前先比较项目本地修改和中央库内容。

## 重要区别

`项目工作区` 不是中央库本身，它是项目本地状态的对比和流转界面。
