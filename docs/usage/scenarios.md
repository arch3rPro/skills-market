# Scenarios

## Purpose

`Scenarios` let you maintain separate enabled skill sets for different workflows.

## Why They Matter

Without scenarios, every workflow shares the same active skills, which quickly becomes noisy. Scenarios let you isolate contexts such as:

- personal coding
- client work
- learning experiments
- product-specific agent setups

## What You Can Do

- Create new scenarios.
- Rename scenarios.
- Reorder scenarios.
- Switch active scenario from the sidebar.
- Keep the same central library while changing the active enabled set.

## How They Interact With Skills

- The central library stores all imported skills.
- A scenario decides which of those skills are enabled right now.
- Sync results depend on both the current scenario and each skill's per-agent sync state.

## Recommended Use

Create scenarios based on workflow boundaries, not minor preferences. A small number of clear scenarios is easier to maintain than many overlapping ones.
